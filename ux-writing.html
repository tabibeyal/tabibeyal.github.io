Microcopy Observations

A growing collection of interface labeling problems found in real products — each analyzed using a What / Problem / Suggestion structure.

Format What → Problem → Suggestion (per observation) Scope Button labels, placeholder text, icon tooltips, navigation consistency Languages Hebrew, English
01 — Microcopy Observations

Single-element problems

Each observation focuses on one UI element — a button, a search field, an icon — where the label creates confusion, inconsistency, or unnecessary friction.

Israel Meteorological Service Toggle button
What
A forecast paragraph is cut off mid-text. A button labeled "מידע נוסף" (Additional Information) reveals the rest. Once expanded, it changes to "הראה פחות" (Show Less).
Problem
"Additional Information" implies separate, supplementary content — not the continuation of the same paragraph. The two button states are not a logical pair: "Additional Information" and "Show Less" follow different naming patterns.
Suggestion
Replace "מידע נוסף" with "הצג יותר" (Show More), creating a consistent pair: "הצג יותר" / "הצג פחות" (Show More / Show Less).
Alison.com Search fields
What
Two search fields serve the same function. The navigation bar reads "What do you want to learn" while the main area reads "What do you want to learn today?"
Problem
Inconsistent wording signals that the fields might behave differently. The dominant element encourages browsing for a new course, while "Continue Learning" — the likely goal for returning users — sits small at the edge of the navigation bar.
Suggestion
Use identical placeholder text in both fields. For logged-in users, give "Continue Learning" visual priority — as the central call to action or with greater prominence in the header.
AliExpress Sidebar icons
What
A floating sidebar has four icon buttons with no descriptive labels. The top button shows a promotional tooltip on hover; the remaining three have no hover response at all.
Problem
Icons without text labels force users to guess their function. The first button's hover tooltip sets an expectation the others should respond too — but they don't. Users must click each button to discover what it does.
Suggestion
Add descriptive hover tooltips to all four buttons — one or two words naming the function. Keep promotional content separate from navigation labels.
02 — Navigation Audits

System-level labeling problems

These observations map how inconsistent labeling across multiple entry points creates confusion at the navigation level — not just within a single element.

Tel Aviv University — Registration Entry Points
Homepage navigation
What
The homepage offers four separate links to the same registration form, each worded differently:
LabelLocation
"רישום לאוניברסיטה" (black button)Top navigation
"רישום לאוניברסיטה" (text link)Secondary navigation
"הרשמה" (Registration)Upper strip
"הרשמה ללימודים" (Registration for Studies)Footer
The form itself presents a button labeled "כניסה" (Enter/Login) — a label that does not describe the action a first-time applicant is taking.
Problem
The labels alternate between "רישום" and "הרשמה" (both meaning registration) — two Hebrew words for the same action — and vary in specificity. A prospective student may wonder whether each link leads somewhere different. The form's "כניסה" button adds ambiguity: it typically means login, not an action a first-time applicant would associate with starting registration.
Suggestion
  • Standardize all registration links to a single label — either "רישום לאוניברסיטה" or "הרשמה לאוניברסיטה" (both meaning University Registration), consistently.
  • Replace "כניסה" on the form with a label that describes the action, such as "המשך להרשמה" (Continue to Registration) or "שלח פרטים" (Send Details).
Tel Aviv University — Contact Entry Points
Homepage navigation & dropdown menu
What
A dropdown menu under "אפשר לעזור לך?" organizes contact options into two categories with five links. Outside the dropdown, several contact options appear independently:
Label on homepageDestination
"דברו איתנו" (Talk to Us)Contact page with form and map
"אנחנו כאן לייעץ לך!" (We're Here to Advise You!)Marketing page about advising
"היי, כיצד נוכל לסייע לך?" (Hi, How Can We Help?)Live chat
Within the dropdown, two links in separate categories lead to the same external scheduling tool. The same contact form appears in three locations. The tone shifts from formal to promotional to casual across entry points.
Problem
The dropdown provides a clear structure, but the rest of the homepage undermines it by duplicating destinations under different labels. A user who finds "אנחנו כאן לייעץ לך!" has no reason to know it leads to the same page as "סיוע בבחירת מסלול לימודים" in the dropdown. The tone shifts between entry points suggest different services, when in practice several reach the same destination.
Suggestion
  • Remove duplicate links outside the dropdown, or align their wording with the dropdown labels.
  • Consolidate the two paths to the scheduling tool into a single link within the dropdown.
  • Add a direct link to the scheduling tool on the advising page.
  • If the embedded form at the bottom of the homepage serves a purpose, add an anchor link from the navigation.
  • Align the tone across all contact touchpoints — for example: "דברו איתנו" (Talk to Us), "איך עובד הייעוץ האישי?" (How Does Personal Advising Work?), "צריכים עזרה?" (Need Help?).
